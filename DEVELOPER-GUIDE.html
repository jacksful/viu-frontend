Getting Started

Prerequisites: Node.js 18+ and npm

1
Install dependencies 
cd VIU
npm install

This recreates the node_modules/ folder (~900MB). This is normal.

2
Configure environment variables 
cp .env.local.example .env.local
# Or edit .env.local directly
3
Run the development server 
npm run dev
# Opens at http://localhost:3000
4
Build for production 
npm run build
npm start
Note: Do NOT include node_modules/ or .next/ when sending files. They are auto-generated. The .gitignore already excludes them.
Section 4 — Important

Laravel API Integration

The website has 2 forms that send data to your Laravel dashboard via API proxy routes. Here's how they work:

How It Works

User fills form on Next.js site
        ↓
POST request to Next.js API route (/api/zip-check or /api/contact)
        ↓
Next.js forwards request to your Laravel API
        ↓
Laravel processes data → stores in dashboard
        ↓
Response sent back to user

Form 1: ZIP Code Availability Check

PropertyValue
Next.js Route/api/zip-check
Filesrc/app/api/zip-check/route.ts
MethodPOST
Request Body{ "zipCode": "90210" }
Expected Response{ "available": true, "message": "..." }

Your Laravel endpoint should:

  • Accept POST with JSON body { "zipCode": "90210" }
  • Check if the ZIP is available in your database
  • Return { "available": true/false, "message": "..." }

Form 2: Lead Capture / Territory Claim

PropertyValue
Next.js Route/api/contact
Filesrc/app/api/contact/route.ts
MethodPOST
Request Body{ "name", "email", "phone", "company", "zipCode", "source" }
Expected Response{ "success": true, "message": "..." }

Full request body example:

{
  "name": "John Smith",
  "email": "john@example.com",
  "phone": "555-0123",
  "company": "RE/MAX Elite",
  "zipCode": "90210",
  "source": "territory-claim"
}

Laravel Endpoint Examples

// routes/api.php

Route::post('/zip-check', function (Request $request) {
    $zipCode = $request->input('zipCode');

    $exists = Territory::where('zip_code', $zipCode)
                       ->where('active', true)
                       ->exists();

    return response()->json([
        'available' => !$exists,
        'zipCode'  => $zipCode,
        'message'  => $exists
            ? "ZIP {$zipCode} is currently taken."
            : "ZIP {$zipCode} is available!",
    ]);
});

Route::post('/contact', function (Request $request) {
    $validated = $request->validate([
        'name'    => 'required|string',
        'email'   => 'required|email',
        'phone'   => 'nullable|string',
        'company' => 'nullable|string',
        'zipCode' => 'nullable|string|size:5',
        'source'  => 'nullable|string',
    ]);

    Lead::create($validated);

    return response()->json([
        'success' => true,
        'message' => 'Territory specialist will contact you within 24 hours.',
    ]);
});

Connecting Your Laravel API

Edit .env.local in the VIU project root:

# Your Laravel API base URLs
ZIP_CHECK_API_URL=https://fullviu.com/api/zip-check
CONTACT_API_URL=https://fullviu.com/api/contact

# If your Laravel API uses Bearer token auth
API_KEY=your-laravel-api-key-here
CORS: Your Laravel app needs to allow requests from the Next.js domain. Add it to config/cors.php → allowed_origins.

Section 6 — Optional

Sanity CMS Setup

Sanity lets non-developers edit content via a visual dashboard at /studio. This is optional — you can edit files directly instead.

1
Create a Sanity project 
npx sanity init --project-id <generated>

Or create one at sanity.io/manage

2
Add credentials to .env.local 
NEXT_PUBLIC_SANITY_PROJECT_ID=your-project-id
NEXT_PUBLIC_SANITY_DATASET=production
3
Access the CMS

Go to http://localhost:3000/studio — the content editor is embedded in the site.

Schemas are pre-configured in sanity/schemas/ for all page sections.

Section 7

Adding New Pages

The project supports dynamic pages via src/app/[slug]/page.tsx.

Option A: Create a static page

Add a new file at src/app/about/page.tsx:

export default function AboutPage() {
  return (
    <div className="mx-auto max-w-[1440px] px-5 py-20">
      <h1>About VIU</h1>
      <p>Your content here...</p>
    </div>
  );
}

It's instantly available at /about.

Option B: Create via Sanity CMS

Create a new "Page" document in Sanity Studio, add sections, set the slug — it auto-renders at /your-slug.

Section 8

Deployment Guide

Option A: Vercel (Recommended — Free)

1
Push code to GitHub/GitLab
2
Go to vercel.com → Import project
3
Add environment variables in Vercel dashboard (same as .env.local)
4
Deploy — Vercel auto-detects Next.js. Zero config needed.
5
Add custom domain (e.g., fullviu.com) in Vercel → Settings → Domains

Option B: Hostinger VPS / Any Node.js Server

# On the server:
git clone your-repo
cd VIU
npm install
npm run build

# Start production server (use PM2 for persistence)
npm install -g pm2
pm2 start npm --name "viu" -- start
pm2 save
Important: Set environment variables on the server before building. Either create .env.local on the server or set them via your hosting panel.

Option C: Static Export (Shared Hosting)

Add to next.config.ts:

const nextConfig = {
  output: 'export',
};
npm run build
# Upload the `out/` folder to Hostinger shared hosting
Note: Static export disables API routes. Forms would need to call your Laravel API directly from the client instead of proxying through Next.js.
